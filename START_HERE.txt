════════════════════════════════════════════════════════════════
  🔑 AUTHENTICATION OPTIONS
════════════════════════════════════════════════════════════════

Option 1: Personal Access Token (PAT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ Easier to set up
✓ Works right away
✗ Expires after 30 days
✗ Need to generate new token each time

Steps:
1. Go to: https://github.com/settings/tokens/new
2. Create token with 'repo' scope
3. Copy token
4. Paste as password when git asks

Option 2: SSH Key (Recommended)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ One-time setup
✓ Never expires
✓ More secure
✗ Slightly more complex

Steps:
1. Generate: ssh-keygen -t ed25519
2. Add public key to GitHub: https://github.com/settings/ssh
3. Test: ssh -T git@github.com
(Already documented in GITHUB_PUSH_GUIDE.md)

Q: What if I get "fatal: not a git repository"?
A: Run init-git-bizeng.bat again to initialize.

Q: What if GitHub asks for credentials repeatedly?
A: Set up SSH key (see GITHUB_PUSH_GUIDE.md) or use PAT.

Q: What if the repository URL is wrong?

   (Check if this URL exists before running)

Q: Can I run the script multiple times?
A: Yes! It safely removes old .git and starts fresh.
